Use Case: Manager maintaining a location on a store by store basis
--------------------------------------------------
CHARACTERISTIC INFORMATION
Goal in Context: To allow Managers to easily maintain a brick and morter store and it's issues
Scope: Manager store interaction
Level: Primary task
Preconditions: Manager is at a store location and the store is open
Success End Condition: All issues resolved
Failed End Condition: Store loses a customer or system
Primary Actor: Manager
Trigger: Employee or store has an issue
----------------------------------------
MAIN SUCCESS SCENARIO
1. Server has an issue that cannot be resolved
2. Manager attends to issue
3. Issue is resolved technically or through customer service
----------------------
RELATED INFORMATION (optional)
Priority: Top priority
Performance Target: All issues resolved
Frequency: Daily or per occurence
Subordinate Use Cases: Ice cream server sale transaction
Channel to primary actor:  Ice cream server or customer to manager
Secondary Actors: Ice cream server, Customer
Channel to Secondary Actors: customer to server or system to server
----------------------------
